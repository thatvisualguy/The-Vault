03
FAILURE #1
Solving a problem
nobody has.
Teams get excited about AI and retrofit it onto processes that don't need it. The use case sounds impressive in a boardroom. It solves nothing in the real world.
WHAT THIS LOOKS LIKE
"We built an AI document classifier for a team that processed 12 docs a month. The manual process took 40 minutes. The AI project took 8 months to build."
The fix: Define the problem first. Then ask if AI is actually the right tool — not the other way around.

04
FAILURE #2
No one defined
what success
looks like.
Companies invest six figures into AI with zero definition of ROI. Twelve months later, nobody can answer whether it worked.
✕

No baseline metrics set

If you don't know where you started, you can't prove you moved.

✕

No success threshold agreed

"Better" is not a KPI. Define the number before you build anything.

✓

Define ROI before kickoff

Time saved × hourly rate. Revenue uplift. Cost per decision. Pick one.

05
FAILURE #3
Scope creep
kills AI projects.
"While we're building this, can we also add…" is the most dangerous sentence in an AI project. Bad scoping turns a 3-month build into an 18-month disaster.
✕
WHAT BAD SCOPING LOOKS LIKE
A chatbot project that started with 1 use case grew to 14 use cases before launch. It shipped 22 months late. None of the 14 worked well.
→

Start with the smallest version that proves value. Expand only after you have evidence it works.
